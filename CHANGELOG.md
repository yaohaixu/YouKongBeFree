# Changelog

所有重要变更都会记录在此文件中。格式参考 [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)，版本号遵循语义化版本思路。

## [0.9.0] - 2026-07-10

### Added

- 新增活动可选结束时间 `endsAt`，用于跨天活动展示和自动归档判断。
- 新增 GitHub Actions CI，在 `dev` / `main` push 和 PR 时运行依赖安装、`npm test` 与 `npm run build:cloudbase`。
- `npm test` 新增跨天活动未结束归档的 API 冒烟覆盖。

### Changed

- 活动列表、活动详情、报名确认、报名表、审核待办和后台活动管理统一展示活动起止时间。
- 活动自动结束逻辑优先使用 `endsAt` 判断归档日期；未填写结束时间时继续使用开始日期。
- 静态资源版本参数升级为 `v=0.9.0`，云函数构建版本升级为 `0.9.0`。

### Fixed

- 修复跨天活动仅按开始日期判断时可能被提前归档的问题。

## [0.8.0] - 2026-07-10

### Added

- 新增 `activities.html` 独立活动列表页，默认展示所有近期活动，`?view=history` 展示历史活动。
- 新增活动自动结束 sweep：发布 / 满员活动在活动日期次日 0 点后自动改为「活动结束」。
- 新增系统操作日志动作 `activity.auto_end`，记录自动归档的活动。
- 新增 CloudBase 云函数入口兜底 sweep，线上活动列表请求前会按节流策略检查过期活动。
- `npm test` 新增过期活动自动归档、近期 / 历史活动列表和新增页面移动端无横向溢出覆盖。

### Changed

- 首页「近期活动」移动到「我们是谁」之前，最多展示 3 条活动。
- 首页 Hero「发起一个活动」按钮改为「参加活动」，跳转近期活动列表页。
- 首页活动区新增「查看所有近期活动」和「查看历史活动」入口。
- 公开活动列表默认只返回 `published` / `full` 的未结束活动；历史活动通过 `view=history` 单独查询。
- 静态资源版本参数升级为 `v=0.8.0`，云函数构建版本升级为 `0.8.0`。

### Fixed

- 修复活动结束后仍可能继续出现在首页和近期活动列表的问题。
- 修复固定日期测试样例随时间推移会变成过期活动的隐患。

## [0.7.0] - 2026-07-10

### Added

- 新增统一 `store.query()` 查询接口，JSON 本地模式和 CloudBase 模式共享筛选、排序和分页语义。
- 新增 `npm test`，自动运行语法检查、API 冒烟和 Playwright 浏览器冒烟。
- 新增 `tests/smoke.test.js`，覆盖登录安全头、成员/协作员新增、活动提审、双岗审核、报名、重复报名、报名表、日志查询、报名人数排序、移动端无横向溢出和审核封面图展示。
- 新增 `docs/cloudbase-indexes.md`，记录 CloudBase 推荐索引和查询注意事项。

### Changed

- CloudBase 模式下活动、成员、模块和操作日志列表改为使用 `where`、`orderBy`、`skip`、`limit` 和 `count` 在数据库查询层分页。
- 活动记录新增并维护 `registrationCount`，支持按报名人数进行数据库层排序。
- 协作员列表、成员管理、模块管理、活动管理和操作日志查询统一改为存储层筛选。
- 云函数构建版本升级为 `0.7.0`。

### Fixed

- 修复 CloudBase Store 需要读取集合前 1000 条后再在云函数内筛选分页的性能隐患。
- 修复发版前缺少一键回归测试的问题。

### Removed

- 移除 `lib/app.js` 中已不再使用的旧内存分页和关键词筛选 helper。

## [0.6.0] - 2026-07-10

### Added

- 新增 `registrations.html` 独立报名表详情页，活动发起人和管理员可查看报名者列表、删除报名记录并导出 CSV。
- 新增 `admin-logs.html` 管理员操作日志页，记录登录、退出、新增、保存、删除、提交、审核、退回、拒绝、撤回、报名、取消报名、取消活动和结束活动等关键动作。
- 新增管理员取消活动和结束活动 API 与前端操作按钮。
- 新增活动、成员、模块和日志 API 分页响应，前端「加载更多」改为请求下一页数据。
- 新增操作日志集合 `logs` / `yk_logs`。

### Changed

- 活动、成员、模块和日志筛选改为点击「筛选」后才请求，不再输入时即时搜索。
- 成员角色从成员 / 协作员多选改为单选下拉，YKadmin 仍为系统唯一管理员。
- 活动人数限额留空默认 99 人，且最大限制为 99 人。
- 审核意见下拉默认显示不可提交的「请选择」。
- 报名表从活动搜索 / 列表页移出，改为单独详情页承载。
- 优化登录页、筛选面板、工具栏、表单按钮和移动端日期输入的间距与宽度约束。
- 静态资源版本参数升级为 `v=0.6.0`。

### Fixed

- 修复移动端 `datetime-local`、开始日期和结束日期输入可能撑出页面边距的问题。
- 修复报名表混在活动列表中导致列表臃肿的问题。
- 修复前端“加载更多”通过不断扩大 `pageSize` 取数的隐患，避免数据超过后端单页上限后卡住。
- 修复只有成员角色时仍可能看到无意义待办入口的延续问题，工作台继续按角色显示入口。

### Removed

- 移除旧的内联报名表面板逻辑。
- 移除成员角色多选控件和旧多选样式。

## [0.5.0] - 2026-07-10

### Added

- 新增入口型「我的工作台」，成员先看活动概览，再进入独立功能页。
- 新增 `activity-editor.html` 发起 / 编辑活动页面。

- 新增 `review-tasks.html` 审核待办页面，管理员和协作员按当前审核岗查看自己的待办。
- 新增入口型 `admin.html` YKadmin 工作台。
- 新增 `admin-activities.html` 全部活动管理页，支持关键词、模块、状态、日期和排序筛选。
- 新增 `admin-members.html` 成员管理页，支持关键词和角色筛选。

- 新增 CloudBase 静态构建清单中的管理子页面，避免线上入口跳转 404。

- 后台从单页长表单调整为工作台入口 + 子页面管理结构。
- 首页、活动与参与页、活动详情页等发起活动入口统一指向新的 `activity-editor.html`。
- 静态资源版本参数升级为 `v=0.5.0`。

### Fixed

- 修复新增子页面未进入 CloudBase Hosting 静态构建清单的问题。
- 修复 `.button` 等样式覆盖 HTML `hidden` 属性，导致「取消」和「再加载 0 条」等隐藏控件外露的问题。

## [0.4.3] - 2026-07-10

### Added

- 新增安全响应头：CSP、`X-Frame-Options`、`X-Content-Type-Options`、`Referrer-Policy`、`Permissions-Policy`，HTTPS 环境返回 HSTS。
- 新增所有 HTML 页面的 `Content-Security-Policy` meta 和 referrer meta，补齐 CloudBase Hosting 静态页的基础浏览器侧约束。
- 新增非 GET API 请求安全校验头要求，前端统一发送 `X-Requested-With: XMLHttpRequest`。
- 新增登录、报名和普通写操作的内存级限流。
- 新增 `docs/security.md`，记录当前安全控制和遗留风险。
- 新增 `CORS_ORIGINS` 和 `SESSION_MAX_AGE_DAYS` 环境变量说明。

### Changed

- Session 服务端存储从明文 token 调整为 token 哈希，并增加过期时间。
- 登录 token 随机强度提升到 32 字节；新生成 ID 的随机后缀提升到 8 字节。
- CORS 允许头从回显请求头调整为固定白名单。
- 公开协作员接口和登录态接口不再返回手机号，管理员成员管理接口仍返回手机号。
- 上传封面限制为 JPG、PNG、WebP、GIF，单文件最大 6MB。
- 手机号、昵称、模块、活动标题、地点、描述和审核说明增加格式与长度校验。
- 静态资源版本参数升级为 `v=0.4.3`。

### Fixed

- 修复上传错误和请求体过大时可能返回泛化 500 的问题，改为明确 4xx 错误。
- 降低跨站表单提交、点击劫持、内容嗅探和过度暴露成员手机号的风险。

### Removed

- 无。
