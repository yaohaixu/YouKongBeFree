# Changelog

所有重要变更都会记录在此文件中。格式参考 [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)，版本号遵循语义化版本思路。

## [0.16.3] - 2026-07-14

### Changed

- 分享海报标题和字段值移除 `【】` 装饰符号，标题直接显示为「模块丨标题」。
- 分享海报地址和日期改为单行展示，地址严格使用活动填写地点，日期精确到完整年月日时分，例如 `2026年7月12日20:00-2026年7月12日23:00`。
- 分享海报二维码说明文字移动到二维码上方，并删除底部明文活动网址。
- 静态资源版本参数升级为 `v=0.16.3`，项目版本升级为 `0.16.3`。

### Fixed

- 修复分享海报将需求说明里的括号符号误当成正式视觉文案的问题。

## [0.16.2] - 2026-07-14

### Changed

- 审核待办页 PC 端「审核意见」「备注」「提交审核」调整为统一审批面板，选择框、备注框和按钮底部对齐，移动端维持原有单列布局。
- `npm run test:visual` 增加一条临时审核待办数据，使审核待办截图可以覆盖真实审批操作区。
- 静态资源版本参数升级为 `v=0.16.2`，项目版本升级为 `0.16.2`。

### Fixed

- 修复 PC 端审核待办审批意见和备注区域视觉重心不齐的问题。

## [0.16.1] - 2026-07-14

### Added

- 发起活动页新增「是否展示活动发起人联系方式」字段，选择「是」后展示联系方式输入框，默认带出当前登录手机号并允许修改。
- 活动公开详情页在发起人选择展示时显示「发起人联系方式」。
- 新增 `/api/qr` SVG 二维码接口，用于分享海报生成活动二维码。
- 报名成功页新增「下载分享海报」按钮，海报可带入报名昵称和手机号。

### Changed

- 分享海报改为按封面原始比例完整展示封面，不再裁切长图。
- 分享海报重新排版：展示「模块丨标题」、发起人、诚邀昵称、报名手机号、地址、日期和右下角活动二维码，地址与日期分行显示；地点为空或仅为「有空客厅」时默认展示「有空客厅|江北劳动一村」。
- 当前登录用户的 session / login 响应返回本人手机号，用于发起活动页默认联系方式。
- 静态资源版本参数升级为 `v=0.16.1`，项目版本升级为 `0.16.1`。

### Fixed

- 避免关闭「展示发起人联系方式」时向公开活动 payload 泄露联系方式字段。
- 优化分享海报二维码区域，避免二维码与文字重叠。

## [0.15.0] - 2026-07-13

### Added

### Fixed

- 修复富文本编辑器只能挂载活动描述 `textarea[name="description"]`，无法复用到模板内容编辑框的问题。
- 修复无扩展名图片文件上传正文图时可能因文件名缺少 `.jpg` 后缀被服务端拒绝的问题。
